# Central Store — ระบบสั่งซื้อ/เบิกสโตร์กลาง + คู่มือการใช้งาน

ระบบบริหารสโตร์กลางของโรงแรม แบบ "อนุมัติครั้งเดียวจบ" พร้อมวงจรบัญชีเต็มรูปแบบ
หน้าเว็บ (GitHub Pages) → API + engine บัญชี (Google Apps Script) → ฐานข้อมูล (Google Sheets) ทั้งหมดอยู่บน free tier

```
GitHub Pages (index.html)  ──fetch──►  Apps Script Web App (Code.gs)  ──►  Google Sheets
```

---

## วงจรเอกสาร & บัญชี (4 จังหวะ)

| จังหวะ | ใคร | บันทึกบัญชี |
|---|---|---|
| 1. สั่งซื้อ + **อนุมัติ** | จัดซื้อ / ผู้อนุมัติ | ยังไม่ลงบัญชี (เป็นการผูกพันงบ) — เซ็นต์/อนุมัติ **ครั้งเดียว** |
| 2. **รับของ (GRN)** | สโตร์ | Dr สินค้าคงเหลือ *(หรือ ค่าใช้จ่ายแผนก ถ้าของสด)* / Cr พักรับของ GR/NI |
| 3. **ใบกำกับ (Invoice)** | บัญชีเจ้าหนี้ | Dr พักรับของ GR/NI + Dr ภาษีซื้อ (VAT) / Cr เจ้าหนี้การค้า |
| 4. **เบิกออก** | แผนกที่ใช้ | Dr ค่าใช้จ่ายแผนก (ต้นทุนถัวเฉลี่ย) / Cr สินค้าคงเหลือ |

หลักสำคัญ
- **อนุมัติแยกจากรับของแยกจากใบกำกับ** (3-way match: PO ↔ GRN ↔ Invoice)
- ของเข้าคลัง = **สินทรัพย์** ยังไม่ใช่ค่าใช้จ่าย จะเป็นค่าใช้จ่ายตอน **เบิก** เท่านั้น
- **VAT รับรู้ตอนได้ใบกำกับภาษี** (จังหวะ 3) ไม่ใช่ตอนรับของ — เก็บเลขที่ใบกำกับไว้ทำรายงานภาษีซื้อ
- บัญชีพัก **GR/NI** = มูลค่าของที่รับมาแล้วแต่ supplier ยังไม่วางบิล (ยอดนี้ควรเป็น 0 เมื่อวางบิลครบ)
- ของสด (ผัก/ของทะเล) = **ตัดจ่ายตรง** เข้าแผนกตอนรับ ไม่เก็บสต็อก
- ต้นทุน = **ถัวเฉลี่ยถ่วงน้ำหนัก**

## ส่วนที่ 1 — ติดตั้ง Backend (Google Sheets + Apps Script)

1. สร้าง Google Sheet ใหม่ (sheets.new) เช่น `Central Store DB`
2. **Extensions → Apps Script** → ลบโค้ดเดิม → วางทั้งหมดจากไฟล์ `Code.gs` → **Save**
3. เลือกฟังก์ชัน `setup` แล้วกด **Run** (ครั้งแรกต้อง Authorize ด้วยบัญชี Google)
   - กลับมาดู Sheet จะมีชีท Items, PurchaseOrders, Receipts, Invoices, Journal ฯลฯ พร้อมข้อมูลตัวอย่าง
4. **Deploy → New deployment → Web app** → Execute as: **Me** / Who has access: **Anyone** → **Deploy**
5. คัดลอก **Web app URL** (ลงท้าย `/exec`)

> ⚠️ ทุกครั้งที่แก้ `Code.gs` ต้อง **Deploy → Manage deployments → Edit → Version: New version** ไม่งั้น URL เดิมยังรันโค้ดเก่า
> ⚠️ เวอร์ชันนี้ (v3) เพิ่มชีท Invoices ใหม่ — ถ้าอัปจากเวอร์ชันก่อน ต้องรัน `setup()` ใหม่อีกครั้ง

---

## ส่วนที่ 2 — ติดตั้ง Frontend (GitHub Pages)

1. สร้าง repo เช่น `central-store` (Public)
2. Upload `index.html` เข้า root (ไฟล์ฝัง Web app URL ไว้แล้ว ถ้า deploy ใหม่ได้ URL ใหม่ ค่อยกดเฟืองในแอปเปลี่ยน)
3. **Settings → Pages → Deploy from a branch** → `main` / `/(root)` → Save
4. เปิด URL เช่น `https://<user>.github.io/central-store/` — ใช้งานได้เลย

## ส่วนที่ 3 — คู่มือการใช้งาน (รายหน้าจอ)

### หน้า ภาพรวม
ศูนย์รวมงานค้าง แบ่งเป็น 3 โซนตามจังหวะงาน
- **คำขออนุมัติ** — PO ที่เกินวงเงิน กด *อนุมัติ (ครั้งเดียวจบ)* แล้วตกไปโซนรับของ / *ปฏิเสธ* ได้
- **รอรับเข้าคลัง** — กรอกจำนวนที่รับจริงต่อรายการ (รับไม่ครบได้ → สถานะ "รับบางส่วน") กด *บันทึกรับเข้าคลัง (GRN)* → สต็อกเพิ่ม + ลง GR/NI
- **รอวางบิล/ใบกำกับ** — เมื่อ supplier ส่งใบกำกับภาษีมา: กรอกจำนวนที่บิล, **เลขที่ใบกำกับภาษี**, และยอด **VAT** (ระบบเติม 7% ให้ แต่แก้ให้ตรงบิลจริงได้) กด *บันทึกใบกำกับ & ตั้งหนี้* → ล้าง GR/NI + รับรู้ VAT + ตั้งเจ้าหนี้
- การ์ด KPI: มูลค่าคลัง / ค่าใช้จ่ายเข้าแผนก / ภาษีซื้อสะสม / รออนุมัติ + ป้าย **GR/NI** (ของรับแล้วยังไม่วางบิล)
- ตั้ง **เกณฑ์อนุมัติอัตโนมัติ** ได้ (PO ไม่เกินวงเงิน = ระบบอนุมัติเอง)

### หน้า สั่งซื้อ
เลือก Supplier → เพิ่มรายการ (เลือกของ, จำนวน, ราคา/หน่วยก่อน VAT) → ถ้าเลือกของสดจะบังคับเลือกแผนก
แถบสีบอกล่วงหน้าว่าจะ *อนุมัติอัตโนมัติ* หรือ *ต้องส่งอนุมัติ* → กดส่ง

### หน้า เบิก
เลือกแผนก (cost center) → เลือกของจากคลัง + จำนวน → *บันทึก & ตัดสต็อกทันที* ไม่ต้องอนุมัติ ต้นทุน (ถัวเฉลี่ย) วิ่งเข้าแผนกนั้น

### หน้า คลัง
ดูสินค้าคงเหลือ + ต้นทุนถัวเฉลี่ย + มูลค่า · ส่วน **นับสต็อกสิ้นเดือน** กรอกยอดนับจริง → ระบบลงผลต่าง (variance) เข้าบัญชีให้

### หน้า บัญชี
- **สมุดรายวัน** — กดดูคู่ Dr/Cr ของทุกรายการ (ยอดยกมา, รับของ, ใบกำกับ, เบิก, นับสต็อก)
- **ต้นทุนตามแผนก** — กราฟค่าใช้จ่ายแยก cost center
- **Audit trail** — log ทุกการกระทำ + เวลา + ผู้ทำ (ใช้แทนลายเซ็นกระดาษ)

### ปุ่มเฟือง (มุมขวาบน)
เปลี่ยน Web app URL หรือ **รีเซ็ตข้อมูล** กลับเป็นชุดตัวอย่าง

## ส่วนที่ 4 — ลองตามสถานการณ์ตัวอย่าง

ข้อมูลตัวอย่างเตรียมไว้ให้ลองครบวงจร
1. ไปหน้า **ภาพรวม** → PO หลอดไฟ LED (PO-2569-003) อยู่ทั้งในโซน "รอรับเข้าคลัง" (เหลือรับ 40) และ "รอวางบิล" (รับมาแล้ว 60 รอบิล)
2. โซนรอวางบิล: กรอกเลขใบกำกับ เช่น `TI680617-001` → กด *บันทึกใบกำกับ* → ดูป้าย GR/NI ลดลง + ภาษีซื้อเพิ่ม
3. โซนรอรับ: รับ 40 ที่เหลือ → ดูต้นทุนถัวเฉลี่ยขยับ + ใบนั้นรับครบ
4. PO ใบใหญ่ของ Makro (PO-2569-004) ค้างอนุมัติ → กดอนุมัติ → ตกไปโซนรับของ → รับ → วางบิล
5. ไปหน้า **บัญชี → สมุดรายวัน** ดู Dr/Cr ของแต่ละจังหวะ

## ส่วนที่ 5 — API (ต่อยอด เช่น เชื่อม LINE OA)

`POST` body เป็น JSON ใช้ `Content-Type: text/plain` (เลี่ยง CORS preflight)

| action | payload | ผล |
|---|---|---|
| `state` | — | คืนสถานะทั้งหมด |
| `submitPO` | `{payload:{supplier, department, date, lines:[{itemId,qty,unitPrice}]}}` | ตั้ง PO |
| `approvePO` | `{no, approver}` | อนุมัติ |
| `rejectPO` | `{no}` | ปฏิเสธ |
| `receivePO` | `{no, lines:[{itemId,qty}]}` | รับของ (Dr คลัง/Cr GR/NI) |
| `invoicePO` | `{no, supplierInvNo, lines:[{itemId,qty}], vat}` | ใบกำกับ (Dr GR/NI+VAT / Cr เจ้าหนี้) — `vat` ว่าง=คิด 7% ให้ |
| `issue` | `{payload:{department, date, lines:[{itemId,qty}]}}` | เบิก + ตัดสต็อก |
| `count` | `{itemId, counted}` | ปรับผลนับ |
| `setThreshold` | `{value}` | ตั้งวงเงินอนุมัติอัตโนมัติ |
| `reset` | — | รีเซ็ตข้อมูลตัวอย่าง |

ทุก action (ยกเว้น state) คืนสถานะใหม่ทั้งก้อนกลับมา

## ส่วนที่ 6 — ข้อควรรู้ก่อนใช้จริง

- เลือกวิธีตีราคา (ที่นี่ถัวเฉลี่ยถ่วงน้ำหนัก) ให้คงที่ตลอดงวด
- ของจริง VAT ต้องตรงกับใบกำกับภาษีของ supplier เป๊ะ (ระบบเติม 7% ให้เป็นค่าเริ่มต้น แต่ควรแก้ให้ตรงบิล)
- บัญชีพัก GR/NI ควรกระทบยอดให้เป็น 0 ทุกสิ้นงวด (ถ้าค้าง = มีของรับแล้วยังไม่ได้ใบกำกับ)
- การใช้ digital approval แทนลายเซ็นกระดาษ ควรเขียน internal control เป็นลายลักษณ์อักษร และเคลียร์กับผู้สอบบัญชี/มาตรฐาน (น่าจะ TFRS for NPAEs) ก่อน
- ต้นแบบเก็บ lines เป็น JSON ในเซลล์เดียวเพื่อความง่าย ถ้าจะทำรายงาน/pivot ใน Sheets เยอะ ค่อย normalize เป็นตารางย่อยภายหลัง
